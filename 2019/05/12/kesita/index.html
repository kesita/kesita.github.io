#Linux高级命令进阶

  • 重定向 > 与 追加重定向 >>
    ○ > 将输出结果写入到文件中（覆盖保存，原来的内容就没了）
    ○ >> 将输出结果写入到文件中（保留原内容并追加在文件最后面）
    ○ 所有有输出结果的指令都能通过重定向写入文件，如：

    echo hello > 123    将hello写入文件
ls ~/123 > 123       将123目录下的文件列表写入文件

    ○ &>:标准和错误输出共同写入文件，会覆盖。
    ○ &>>:标准和错误输出共同写入文件，追加输出，不覆盖。

  • 输入重定向（不常用）
    <

  • 管道符
    前面命令输出的结果，交给后面的命令继续处理

    cat /etc/passwd | wc -l
-21

  • 通配符
    *： 表示匹配0或者多个
    ？： 表示匹配单个字符,该字符必须存在
    [0-9]: 表示匹配0-9之间的一个数字

    [123]: 表示匹配1或者2或者3

    [a-z]: 表示匹配a-z之间的一个字母

    [abc]: 表示匹配a或者b或者c

    [0-9a-z]：匹配到一个字符，是数字或小写字母

  • 引号的作用
    单引号：所见即所得
    双引号：解释变量
    反引号：执行命令
    如： name = ls 加反引号name=ls输出的内容 不加反引号就是name=ls

    ' $name '输出  $name
" $name "输出   xxx （ls的输出）
[root@costa ~] name=costa
[root@costa ~] echo "this is $name"
this is costa
[root@costa ~] echo 'this is $name'
this is $name

    反引号使用：两条命令变成一条命令

    [root@costa ~]which ehco
/user/bin/ehco
[root@costa ~]rpm -pf /user/bin/ehco

[root@costa ~]rpm-pf `which ehco`

  • 软硬连接
    软连接：ln -s 源文件 目标文件

    删除连接文件对源文件无影响
删除源文件连接文件失效
修改源文件连接文件也会跟着变
修改连接文件源文件也会跟着变
源文件被删除后在创建同名源文件，连接文件自动连接

    硬链接：ln 源文件 目标文件

    删除源文件连接文件不会失效
删除连接文件源文件无影响
修改源文件连接文件也会跟着变
修改连接文件源文件也会跟着变
  • Linux磁盘系统
    /root: root用户家目录
    /home/用户名: 普通用户家目录
    /var: 放置经常变动的文件如日志
    /etc 放置服务程序的配置文件
    /usr/local 用户自定义程序的安装位置
    /tmp: 共享临时文件的目录
    /bin，/sbin: 放置二进制命令
    /mnt,/media： 挂载光盘
    /dev: 磁盘设备
